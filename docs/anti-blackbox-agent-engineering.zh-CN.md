# 反黑箱 Agent 工程

AI coding 变危险，不是因为 AI 会改代码，而是因为它变得不透明：agent 改了文件，app 看起来变了，却没人能解释为什么变、怎么恢复。

WishGraph 把可审计性当成一等工程要求。

## Agent 必须解释

执行前：

- 它推断的人类意图是什么。
- 它认为哪些内容在范围内。
- 它不会碰什么。
- 有哪些风险。
- 它将如何验证。

执行后：

- 改了哪些文件。
- 改了哪些行为。
- 哪些检查通过。
- 哪些检查失败或跳过。
- 未来 agent 应该读取哪些状态。

## 项目必须记住

使用持久文件：

- `CODEMAP.md` 用于导航。
- `CONVENTIONS.md` 用于协作规则。
- `ARCHITECTURE.md` 用于边界。
- `tasks/*.md` 用于用户可见的可执行规格。
- `reports/PROJECT_STATUS.md` 用于当前已集成的项目状态概览。

## 人类必须评审正确层级

人类不应该每次都检查每条代码路径。人类应该看到一个压缩 review window，用来判断方向、风险和结果。

## 危险信号

- Agent 没有验证就说 "fixed"。
- 任务没有 "Do Not Do"。
- Agent 改了无关文件。
- 项目地图过期。
- Bug fix 没有识别被污染的 state。
- Agent 依赖 "as discussed above"，而不是文件化规格。

## 核心主张

AI 工程的未来不只是更强模型，而是围绕模型建立更强的项目结构。
