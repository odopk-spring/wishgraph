# 抖音 60-90 秒脚本草案

## 0-5s

AI 写代码最大的问题，不是它不会写代码。

真正的问题是：项目一复杂，它会失忆、乱改、不可追踪。

## 5-20s

我现在用的是一套叫 WishGraph 的方法。

人类不再直接管理每个实现细节，而是表达愿望和做评价。

比如：这个页面更流畅一点；这个结果不对；不要把功能做复杂。

## 20-45s

AI 要做的不是直接开改。

它先把愿望编译成 Spec，再拆成任务，再改代码，再验证，再修复，再写报告。

所有状态都写进项目文件：代码地图、协作规范、架构边界、任务规格、验证报告。

这样换一个会话、换一个 Agent，也能继续。

## 45-70s

调 bug 也不是猜哪个文件。

我要求它按因果链追：

Error -> State -> Code -> Spec。

先找错误现象，再找错误状态，再找代码路径，最后看规范是不是缺了。

## 70-90s

这不是普通多 Agent 调度。

它更像一个意图编译系统：人类许愿，AI 把愿望变成可审计、可验证、可回滚的工程动作。

我会把 WishGraph 的 GitHub 模板和 Codex skill 放出来，后面也会写完整文章。
