# 小红书图文卡片草案

## Card 1

标题：以后人类不再管理所有实现细节

正文：人类只表达愿望和做评价。AI 负责把愿望编译成规格、任务、代码、验证和报告。

## Card 2

标题：传统 AI 写代码的问题

正文：它不是不会写，而是复杂项目会失忆、乱改、不可追踪。上下文一断，项目状态就断。

## Card 3

标题：WishGraph 是什么

正文：低带宽意图 -> Spec -> Task -> Code -> Verify -> Report。把愿望变成可执行、可验证、可回溯的工程图。

## Card 4

标题：不要让聊天窗口当项目记忆

正文：把记忆写进项目：CODEMAP、CONVENTIONS、ARCHITECTURE、任务规格、Dev Report。

## Card 5

标题：反黑箱四问

正文：AI 改了什么？为什么只改这些？怎么验证？失败后从哪里追因果？

## Card 6

标题：调 bug 不再猜文件

正文：Error -> State -> Code -> Spec。先找错误状态，再找代码路径，最后回到规范。

## Card 7

标题：真实案例：PaperChat

正文：我用这个方法把一个复杂本地阅读器拆成讨论 AI、执行 AI、任务规格、代码地图和验证闭环。

## Card 8

标题：开源计划

正文：我会把 WishGraph 做成 GitHub 模板和 Codex skill。文章写思想，仓库放可复用工具。
